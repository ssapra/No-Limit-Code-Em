Sandbox Testing

In order to make sure that you are sending the correct requests and receiving the correct information, you can use this Sandbox Testing routes.


GET /sandbox/current_turn (Get the game's state.)

Syntax

Request
json
{
  "name" : "Joe",
  "game_id" : 43123123,
  "player_key" : 031siodfj23290u32
}
Response
json
{
  "current_player" : "Joe",
  "replacement" : false,
  "hand" : ["5c", "8d", "3d", "As", "Kh"], 
  "bet" : 100,
  "stack" : 400,
  "pot" : 140,
  "min_bet" : 50,
  "max_bet" : 360,
  "max_raise" : 160,
  "betting_summary" : [
    "Sally bets 50",
    "John calls 50"
  ],   
  "replacement_summary" : [
    "Sally replaces 3 cards",
    "John replaces 2 cards"
  ],
  "round_summary" : [
    "John won 200 chips with ["5c", "3d", "8h", "9d", "Ac"] for Hand#16
  ],
  "play" : true,
  "waiting" :false,
  "game_over" : false,
  "message" : "Game is about to Start"
}

POST /sandbox/player_action (Posting an action.)

Syntax

Request
json
{
  "name" : "Joe",
  "game_id" : 43123123,
  "player_key" : 031siodfj23290u32,
  "player_action" : "replce",
  "parameters" : "125"
}
Response
json
{
  [
    "---Received---",
    "Name: Joe",
    "ID: 43123123",
    "Player key: 031siodfj23290u32",
    "Action: replce",
    "Parameters: 125",
    " ",
    "Invalid Action: replce"
  ]
}
